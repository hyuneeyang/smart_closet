# Screen Requirements

## 홈

- 오늘 날씨 요약
- 상황 선택
- 추천 코디 3개
- 추천 이유
- 빠른 액션

## 옷장

- 빠른 등록
- 필터/정렬
- 아이템 수정
- 분석 상태 표시

## 아이템 상세

- 태그 수정
- 착용 기록
- 이 옷으로 코디 추천

## 기록

- 오늘 입은 옷
- 최근 추천 내역
- 피드백 내역

## 설정

- 위치
- 알림
- 스타일 선호
- 추천 강도

## 기능 요구사항

- 사용자는 사진 업로드 또는 링크 입력으로 옷 아이템을 등록할 수 있어야 한다.
- 시스템은 등록된 옷에서 카테고리, 색상, 계절, 보온성, 포멀도를 추출해야 한다.
- 사용자는 추천 전에 상황을 선택할 수 있어야 한다.
- 시스템은 현재 날씨, 체감온도, 강수확률, 바람, 최근 착용 기록을 반영해 코디를 추천해야 한다.
- 시스템은 추천 코디 3개를 제공해야 하며, 각 추천은 서로 충분히 구분되어야 한다.
- 시스템은 각 추천에 대해 추천 이유를 자연어로 설명해야 한다.
- 사용자는 추천 결과에 대해 좋아요, 별로예요, 오늘 입었어요 피드백을 남길 수 있어야 한다.
- 사용자는 특정 아이템을 선택해 해당 아이템 중심으로 코디 추천을 요청할 수 있어야 한다.

## UI/UX 요구사항

- 홈 화면은 앱 진입 직후 오늘 날씨와 추천 코디를 우선 노출해야 한다.
- 상황 선택 UI는 한 손 조작이 쉬운 칩 또는 세그먼트 컨트롤 형태여야 한다.
- 코디 카드에는 이미지, 날씨 요약, 상황 태그, 추천 이유, 액션 버튼이 포함되어야 한다.
- 아이템 등록 플로우는 3단계 이내로 완료 가능해야 한다.
- 추천이 불가능한 경우, 시스템은 단순 오류 대신 원인과 대안을 안내해야 한다.
- 사용자 피드백은 최소 클릭 수로 입력 가능해야 한다.

## 현재 구현 대비 갭

- 홈: 빠른 액션은 일부만 구현, 추천 불가 사유 안내는 보강 필요
- 옷장: 정렬, 분석 상태 배지 보강 필요
- 아이템 상세: 착용 기록, 이 옷으로 코디 추천 미구현
- 기록: 전용 기록 화면 미구현
- 설정: 위치/알림/스타일 선호/추천 강도 설정 화면 미구현
